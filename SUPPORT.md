# Support

This document explains how to get help with Zysk Technologies projects.

## Before You Ask for Help

1. **Check the README** of the specific repository you're working on
2. **Search existing issues** — your question may already be answered
3. **Check the documentation** linked in the repository

---

## For Zysk Team Members

### General Questions & Discussion

- **Slack** — Use the appropriate project channel (e.g., `#proj-tms`, `#engineering`)
- **Microsoft Teams** — For cross-team or management questions

### Technical Issues in Client Projects

1. Raise in the project's Slack channel first
2. If it needs tracking, open a GitHub issue in the relevant repository using the appropriate template

### Infrastructure & DevOps

Contact the engineering lead or raise in `#devops` on Slack.

---

## For External Contributors

If you are an external contributor and have a question that is:

- **A bug** — Open a [Bug Report](https://github.com/zysk/.github/issues/new?template=bug_report.yml)
- **A feature request** — Open a [Feature Request](https://github.com/zysk/.github/issues/new?template=feature_request.yml)
- **A security issue** — See our [Security Policy](SECURITY.md) — do NOT open a public issue

For general questions that don't fit the above, you can open a GitHub Discussion in the relevant repository if Discussions are enabled.

---

## Response Expectations

| Channel | Expected Response Time |
|---------|----------------------|
| Slack (team members) | Same business day |
| GitHub Issues (bugs/features) | 2–5 business days |
| Security reports | Within 48 hours (acknowledgment) |

---

_This document applies to all repositories under the Zysk Technologies organization._
